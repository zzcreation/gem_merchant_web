01 · VISUAL DIRECTION

软萌高幻想，
但仍然是一张策略牌桌。

视觉母题是「兔族宝石王国的商会大厅」：奶油羊皮纸承托界面，五公会的宝石色只用于规则信息。 插画负责氛围，规则组件负责决策 — 两层永远不混。

角色

公会讲述身份

每张发展卡属于五公会之一。公会色环 + 底色渐变让「这张卡是什么颜色的奖励」比看清插画更快。

卡牌

密度随距离切换

桌面用全幅插画卡；手机市场只保留身份芯片（公会色 + 成本 + 声望），插画收进详情抽屉。不是缩小，是换一种读法。

界面

羊皮纸与粉彩天空

柔和的奶油底 + 深绿品牌色承托高对比规则组件；桌面牌桌木质画框只在大屏出现。

原则

插画与规则彻底分层

插画不烘焙数字、边框、文字或状态。声望、成本、可购状态全部由运行时模板绘制。

01

单屏锁定

整局 100svh：状态条、对手行、贵族、三层芯片市场、宝石池、我的状态、行动栏 — 全部同屏。没有页面滚动，没有横向溢出。

02

详情抽屉

点按芯片弹出底部抽屉：全幅插画 + 折扣后成本 + 可购状态，购买 / 预留按钮直接标注 token 得失。支付微调、弃宝石、贵族选择走同一抽屉容器。

03

操作常驻拇指热区

主行动按钮固定底部，文案实时反映选择（「拿宝石 蓝+红」）。所有触控目标 ≥ 44px；不存在悬停依赖。

05 · DESIGN SYSTEM

可直接落到组件的设计规范。

规范延续现有 styles/tokens.css（palette.json 生成），新增布局与密度约定。

TYPE

Baloo 2 + Noto Sans SC

声望、计数等大数字用圆润粗体（自宿主字体子集化）；规则说明与中文正文使用 Noto Sans SC。数字永远比标签大一级。

LAYOUT

100svh 应用壳

游戏内两端都是固定视口应用：height:100svh + 内部 flex/grid 分配，滚动只发生在明确的局部容器（如日志）。

DENSITY

full / chip / mini 三档

同一 CardFace 组件三档密度：桌面市场 full（3:4 全幅插画），手机市场 chip（无插画身份芯片），玩家区 mini（计数点）。按阅读距离切换，不做等比缩放。

RADIUS

8 / 14 / 22 px

徽章与小组件 8；卡牌与面板 14；抽屉与画框 22。软萌但不过度胶囊化。

STATE

描边表达状态

可购 = 绿光晕；需金币 = 金虚线；选中 = 品牌绿粗描边；当前回合 = 金色描边。状态不依赖单一颜色，均附文字或图标。

ART

插画层不含玩法 UI

沿用艺术圣经：不烘焙文字、数字、边框。母版 2:3 不变，容器改为 3:4（full）后 cover 裁上下 — 安全区规则照旧成立。

RESPONSIVE

768 / 1024 两档增强

手机芯片市场为基线；≥768px 换 full 卡 + 横向牌桌；≥1024px 加左右栏（玩家轨 + 行动面板）。只用 min-width。

TOUCH

≥ 44px · 拇指热区

核心行动永远在底部行动栏或抽屉内；顶部只放阅读信息。悬停只作为桌面增强，不承载关键操作。

FINAL REVIEW GATE

统一评审确认后，进入布局重构与全量资产扩展。
