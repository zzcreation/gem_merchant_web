# UI/UX 重设计提案 — 单屏牌桌与密度分层

Status: **PROPOSAL — awaiting review** · 2026-07-25  
Visual page: [`docs/design/ui-redesign/index.html`](./ui-redesign/index.html)（在浏览器中打开，含实时 mockup）  
Reference format: Pass Doudou 统一设计评审（<https://pass-doudou.hezzze.com/design/final-review/>）— 只借鉴其提案结构与「密度分层」思路，视觉沿用本项目艺术圣经（兔拉崩吧 IP、palette.json）。

Approved 后本提案**取代** `docs/ui-overhaul-plan.md` Part A 中的布局假设（5:8 卡框、成本纵条等）；艺术圣经的生成规则（2:3 母版、中心安全区、插画不含 UI）**全部保持不变**。

## 1. 问题（2026-07-24 截图评审）

| # | 问题 | 根因 |
| --- | --- | --- |
| D1 | 桌面需要纵向滚动才能看到三层市场 | 5:8 竖卡 × 3 层的固有高度超过视口；卡高不随视口收缩 |
| D2 | 玩家卡片信息松散、过高 | 玩家面板按移动布局直接放大，没有桌面专用压缩形态 |
| D3 | 桌面背景被裁成近 1:1，横向画框丢失 | 牌桌容器高度由内容撑开（过高），16:9 背景 cover 后左右裁光 |
| M1 | 手机需要滚动到底部才能操作；可横向溢出 | 页面滚动布局 + 无溢出约束；行动面板不固定 |

四个问题同源：**当前布局是「内容撑开页面」的文档流，而牌局 UI 应该是「视口分配空间」的应用壳。**

## 2. 核心方案

### 2.1 单屏应用壳（两端）

游戏内界面锁定 `height: 100svh`，内部用 flex/grid 分配，**无页面滚动、无横向溢出**。滚动只出现在明确的局部容器（日志、抽屉内部）。

### 2.2 密度分层：full / chip / mini（关键决策）

借鉴 Pass Doudou「移动端不缩画，只保留身份」：不同阅读距离渲染不同信息密度，而不是等比缩放同一张卡。

| 档位 | 场景 | 形态 |
| --- | --- | --- |
| `full` | 桌面市场、手机详情抽屉 | **3:4** 全幅插画卡（原 5:8 放弃）；顶部遮幅 = 声望+奖励，底部遮幅 = 成本 |
| `chip` | 手机市场 | 身份芯片：公会色环 + 底色渐变 + 声望 + 奖励点 + 成本徽章；**不含插画** |
| `mini` | 玩家区已购/预留 | 计数点（现有 mini 语义保留） |

母版仍为 2:3；`full` 容器 3:4 时 `object-fit: cover` 改为裁上下（原 5:8 裁左右）。艺术圣经的「主体居中 + 安全边距」规则对两种裁切都成立，**不需要重生成已有资产**。

### 2.3 手机布局（390×844，第一画布）

自上而下（全部同屏）：

1. 状态条（~44px）：标题 · 房间/阶段 · 「轮到你了」
2. 对手行（~52px）：每人一枚横向小胖卡（名字、分、宝石数、预留数；当前回合金描边）
3. 贵族行（~44px）：方形小画框
4. 三层市场（flex-1）：每层 = 纵向 L 标签牌堆 + 4 枚 chip
5. 宝石池（~40px）：6 枚等宽色块，点选态内描边
6. 我的状态（~48px）：分数 + token 计数 + 奖励摘要
7. 行动栏（~64px，拇指热区）：主按钮文案实时反映选择（「拿宝石 蓝+红」）

**详情抽屉（bottom sheet）**：点按 chip 弹出 — 全幅插画 + 折扣后成本 + 可购状态 + 「购买 −绿3 −红2」/「预留 +金1」。支付微调、弃宝石、贵族选择复用同一抽屉容器。

### 2.4 桌面布局（≥1024px 增强）

- 三栏：左玩家轨（双行压缩小卡）· 中横向牌桌 · 右行动面板。
- 牌桌 = 贵族行 + 三层 `full` 卡 + 右缘纵向宝石池，背景用 `bg_merchant_table`（横向画框完整显示）。
- **卡高由视口反推**：`--card-h: clamp(120px, (100svh − chrome) / 3, 220px)` — 任何桌面高度三层全见（解 D1/D3）。
- 玩家小卡两行：`名字+分` / `6 色 token 计数 + 奖励/预留摘要`，当前回合金描边（解 D2）。

### 2.5 状态与规范

- 可购 = 绿描边光晕；需金币 = 金虚线；选中 = 品牌绿粗描边；当前回合 = 金描边。状态均附文字，不依赖单一颜色。
- 字体：Baloo 2（大数字）+ Noto Sans SC（正文）。圆角 8/14/22。断点仍为 768/1024，只用 `min-width`。
- 触控 ≥44px；核心操作固定底部；悬停仅桌面增强。

## 3. 对既有产物的影响

| 产物 | 影响 |
| --- | --- |
| 已促升资产（5 卡 + 贵族 + 桌布 + 宝石 SVG） | **全部继续使用**；卡面改 3:4 容器、桌布回归横向、宝石 SVG 用于宝石池 |
| `CardFace` | `full/compact/mini` → `full/chip/mini`；chip 为新形态，compact（预留购买 pill）并入抽屉 |
| `styles/layout.css` 等 | 游戏内布局重写为应用壳；landing/about/lobby 基本不动 |
| Playwright e2e | 选择器基本可保留（class 不变优先）；快照需重录；新增「无页面滚动/无横向溢出」断言 |
| `docs/ui-overhaul-plan.md` Part A | 布局假设（5:8、成本纵条）废弃；A.3 遮幅法、A.7 `<picture>` 合同、艺术安全区保留 |
| 艺术生产（Wave 2+） | 提示词与母版规格**零改动**；建议评审通过后再开量产 |

## 4. 实施顺序（评审通过后）

1. 手机单屏应用壳 + chip 市场 + 行动栏（先 390×844 验收）
2. 详情抽屉（购买/预留/支付/弃宝石/贵族全部入抽屉）
3. 桌面横向牌桌 + clamp 卡高 + 玩家小卡 + 右行动栏
4. e2e 快照重录 + 溢出断言；CHECKLIST 里程碑 4 动效项衔接
5. Wave 2 L1 × 40 量产

## 5. 待用户确认的问题

1. **chip 不含插画**是否接受？（备选：chip 背景放 12% 透明度插画纹理，成本略增）
2. 手机上**日志**收进「更多」抽屉还是保留一条最新消息条？（提案：状态条只显示最新一条，完整日志入抽屉）
3. 桌面卡框 3:4 是否可接受（比实体牌更方）？（备选：clamp 后保持 2:3，但卡宽收窄，L3 大数字更挤）
