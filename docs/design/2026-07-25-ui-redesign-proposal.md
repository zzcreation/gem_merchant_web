## 1. 问题（2026-07-24 截图评审）

| # | 问题 | 根因 |
| --- | --- | --- |
| D1 | 桌面需要纵向滚动才能看到三层市场 | 5:8 竖卡 × 3 层的固有高度超过视口；卡高不随视口收缩 |
| D2 | 玩家卡片信息松散、过高 | 玩家面板按移动布局直接放大，没有桌面专用压缩形态 |
| D3 | 桌面背景被裁成近 1:1，横向画框丢失 | 牌桌容器高度由内容撑开（过高），16:9 背景 cover 后左右裁光 |
| M1 | 手机需要滚动到底部才能操作；可横向溢出 | 页面滚动布局 + 无溢出约束；行动面板不固定 |

四个问题同源：**当前布局是「内容撑开页面」的文档流，而牌局 UI 应该是「视口分配空间」的应用壳。**

## 2. 核心方案

### 2.1 单屏应用壳（两端）

游戏内界面锁定 `height: 100svh`，内部用 flex/grid 分配，**无页面滚动、无横向溢出**。滚动只出现在明确的局部容器（高级菜单内的统计/日志列表、抽屉内部）。

### 2.2 密度分层：full / simple / mini（关键决策）

不同阅读距离渲染不同信息密度，而不是等比缩放同一张卡。

| 档位 | 场景 | 形态 |
| --- | --- | --- |
| `full` | 桌面市场、手机详情抽屉 | **3:4** 全幅插画卡（原 5:8 放弃）；顶部遮幅 = 声望+奖励，底部遮幅 = 成本 |
| `simple` | 手机市场 | **简化美术卡**：公会色块底 + 简单主题元素 + 运行时叠声望/奖励/成本；点按展开详情抽屉看 `full` |
| `mini` | 玩家区已购/预留 | 计数点（现有 mini 语义保留） |

#### 简化美术（`simple`）规格 — 锁定

- **不是**母版 2:3 的缩小版，也**不是**纯渐变色块。
- **是**为手机市场单独设计的一套图形：以公会主色为色块底，叠少量可读的主题符号 / 剪影 / 纹样（月辉新月与灯笼、潮汐浪纹、林语叶片、余烬炉火、幽夜保险库等），保持软萌高幻想气质。
- 规则信息（声望、奖励点、成本徽章、可购状态描边）仍由**运行时模板**绘制，不烘焙进美术。
- 目标可读尺寸：约 72–96px 宽的手机市场格；图形语言必须在该尺寸下可辨。
- 资产策略（提案偏好，评审可调）：
  1. **优先：按公会 + 等级**做有限套件（5 公会 × L1/L2/L3 = 15 套主题底图），同公会同等级的卡共享简化底，靠奖励色与成本徽章区分个体；或
  2. **备选：按 `artSeed` 一卡一简图**（90 张），表现力更强、制作成本更高。
- 运行时路径建议：`src/assets/cards-simple/<key>.webp`（或 SVG 若主题元素适合矢量）。与全幅 `src/assets/cards/<artSeed>.webp` 并列，由 `CardFace` 的 `simple` 档解析。

全幅母版仍为 2:3；`full` 容器 3:4 时 `object-fit: cover` 裁上下（原 5:8 裁左右）。艺术圣经对全幅卡的安全区规则不变，**已促升的 5 张 L1 全幅继续使用**；简化美术是新增生产线，不要求重做全幅。

### 2.3 手机布局（390×844，第一画布）

自上而下（全部同屏）：

1. 状态条（~44px）：标题 · 房间/阶段 · 「轮到你了」· **高级菜单入口**（⋯）
2. 对手行（~52px）：每人一枚横向小胖卡（名字、分、宝石数、预留数；当前回合金描边）
3. 贵族行（~44px）：方形小画框
4. 三层市场（flex-1）：每层 = 纵向 L 标签牌堆 + 4 枚 `simple` 卡
5. 宝石池（~40px）：6 枚等宽色块，点选态内描边
6. 我的状态（~48px）：分数 + token 计数 + 奖励摘要
7. 行动栏（~64px，拇指热区）：主按钮文案实时反映选择（「拿宝石 蓝+红」）

**详情抽屉（bottom sheet）**：点按 `simple` 卡弹出 — `full` 插画 + 折扣后成本 + 可购状态 + 「购买 −绿3 −红2」/「预留 +金1」。支付微调、弃宝石、贵族选择复用同一抽屉容器。

**高级菜单（Advanced）**：状态条 ⋯ 进入全屏或大半屏面板，**不占主牌桌常驻空间**：

- 对局统计（回合数、各玩家声望/宝石/预留/已购/贵族摘要）
- 完整行动日志（可滚动）
- 房间/设置类入口（复制房间码、回本地、关于等，桌面现有房间菜单能力迁入此处）

主界面**不**常驻日志条；最新反馈仍走现有 StatusBanner / 行动栏文案。

### 2.4 桌面布局（≥1024px 增强）

- 三栏：左玩家轨（双行压缩小卡）· 中横向牌桌 · 右行动面板。
- 牌桌 = 贵族行 + 三层 `full` 卡（**3:4 锁定**）+ 右缘纵向宝石池，背景用 `bg_merchant_table`（横向画框完整显示）。
- **卡高由视口反推**：`--card-h: clamp(120px, (100svh − chrome) / 3, 220px)` — 任何桌面高度三层全见（解 D1/D3）。
- 玩家小卡两行：`名字+分` / `6 色 token 计数 + 奖励/预留摘要`，当前回合金描边（解 D2）。
- 高级菜单（统计 + 日志）在桌面也可从顶栏进入；右栏行动区不塞完整日志。

### 2.5 状态与规范

- 可购 = 绿描边光晕；需金币 = 金虚线；选中 = 品牌绿粗描边；当前回合 = 金描边。状态均附文字，不依赖单一颜色。
- 字体：Baloo 2（大数字）+ Noto Sans SC（正文）。圆角 8/14/22。断点仍为 768/1024，只用 `min-width`。
- 触控 ≥44px；核心操作固定底部；悬停仅桌面增强。

### 2.6 信息架构小结

| 信息 | 手机 | 桌面 |
| --- | --- | --- |
| 市场决策扫描 | `simple` 三层同屏 | `full` 三层同屏 |
| 单卡购买/预留细节 | 详情抽屉（`full`） | 选中卡 + 右栏 / 同抽屉模式可选 |
| 即时反馈 | StatusBanner + 行动栏文案 | 同左 |
| 统计 + 完整日志 | **高级菜单** | **高级菜单**（可选右栏摘要，非必须） |

## 3. 对既有产物的影响

| 产物 | 影响 |

| 已促升全幅资产（5 卡 + 贵族 + 桌布 + 宝石 SVG） | **全部继续使用**于 `full` / 贵族 / 桌布 / 宝石池；卡面容器改 3:4 |

| `styles/layout.css` 等 | 游戏内布局重写为应用壳；landing/about/lobby 基本不动 |
| Playwright e2e | 选择器基本可保留；快照需重录；新增「无页面滚动/无横向溢出」断言 |
| `docs/ui-overhaul-plan.md` Part A | 布局假设（5:8、成本纵条）废弃；A.3 遮幅法、A.7 `<picture>`（全幅）合同、艺术安全区保留；新增 simple 档资产合同 |
| 艺术生产（Wave 2+） | 全幅母版提示词**零改动**；另开简化美术规格与小批量 style-lock 后再量产 |

## 4. 实施顺序（评审通过后 — 尚未开始）

1. 简化美术 style-lock（每公会至少 1 张 L1 样张）→ 人审
2. 手机单屏应用壳 + `simple` 市场 + 行动栏（先 390×844 验收）
3. 详情抽屉 + **高级菜单（统计 + 日志）**
4. 桌面横向牌桌 + clamp 卡高（3:4）+ 玩家小卡 + 右行动栏
5. e2e 快照重录 + 溢出断言；CHECKLIST 里程碑 4 动效项衔接
6. 全幅 Wave 2 L1 × 40 + 简化套件量产（顺序可并行，但 UI 壳先于大批量生产）
